The permanent skillset. These files are loaded by all agents across all projects. They define how to build, generate, analyze, and orchestrate. They do **not** contain client-specific data.

### `01_KNOWLEDGE_NEXUS` — Factual Truth Layer (Always Consulted)
Raw research, NotebookLM exports, PDF drops, client intake documents, market data. This folder is **project-agnostic** — it is the ground truth that all projects draw from. When in doubt, the Nexus has the answer.

**NotebookLM Sync Protocol:**
- Export notebooks as PDF or markdown
- Drop into `01_KNOWLEDGE_NEXUS/` with descriptive filename
- Format: `[TOPIC]_[DATE].md` — e.g., `Arizona_Procurement_Research_2026-04.md`
- Agents will auto-detect and reference on next session load

### `03_JAZZO_AGENCY` — Jazzo Content Manager Project

## ⚠️ RULES FOR ADDING TO THE NEXUS

1. **Only verified factual data goes in `01_KNOWLEDGE_NEXUS`** — no drafts, no speculation
2. **Name files descriptively** — agents use filenames to decide relevance before reading
3. **Date-stamp all new documents** — format: `YYYY-MM` in the filename
4. **Never delete from the Nexus** — archive outdated files in a `_archive/` subfolder instead
5. **Flag contradictions** — if new Nexus data contradicts existing data, create a `-CONFLICT.md` note
